📋 HOW TO USE THE FRAMEWORK
═══════════════════════════════════════════════════════════════════════════════

Workflow Pattern:
1. Read Skill       (Learn why, best practices)
2. Read Prompt      (Learn exact steps)
3. Implement        (Follow + code)
4. Use Agents       (Tests, docs, optimization)
5. Measure          (Verify it works)
6. Document         (Share learnings)

Decision Tree:
  Need new provider?        → Skill: llm-integration
  Need faster/cheaper?      → Skill: performance-optimization
  Need new feature?         → Skill: llm-integration + prompt: add-use-case
  Need tests?               → Agent: @test-generation
  Need docs?                → Agent: @mkdocs-content
  Need understand code?     → Agent: @Explore

💡 IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. Skills BEFORE Prompts
   Always read the skill first to understand WHY before WHAT

2. Agents Are Helpers
   They're powerful but not perfect - review their output always

3. Code Patterns Matter
   All new code should follow existing patterns in:
   - src/main/java/com/example/aispring/service/ProductSearchService.java
   - src/main/java/com/example/aispring/service/SupportService.java

4. Measure Everything
   Every enhancement should include:
   - Metrics (latency, cost, quality)
   - Monitoring (dashboards, alerts)
   - Documentation (what, why, how)

5. Error Handling is Core
   Skill: error-handling/SKILL.md - required reading before implementing anything
