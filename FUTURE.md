Read `performance-optimization/SKILL.md`\n\n---\n\n### Layer 2: Prompts (Step-by-Step)\n**Location**: `/prompts/[action].md`\n\nPrompts are detailed walkthroughs showing exactly what to implement.\n\n**Available**:\n- `add-provider.md` - How to add OpenAI, Claude, etc. (8 steps)\n- `add-use-case.md` - How to create a new feature (10 steps)\n- `optimize-performance.md` - How to improve speed/cost (8 steps)\n- `add-observability.md` - How to add monitoring (10 steps)\n\n**Example**: Ready to add OpenAI? Follow `prompts/add-provider.md`\n\n---\n\n### Layer 3: Agents (AI Automation)\n**Location**: Chat with Copilot\n\nAgents are AI specialists that handle complex work autonomously.\n\n**Available**:\n- `llm-integration` agent - Adds providers automatically\n- `performance-tuning` agent - Optimizes your code\n- `test-generation` agent - Writes tests for you\n- `mkdocs-content` agent - Updates documentation\n- `Explore` agent - Searches codebase\n\n**Example**: Use `@llm-integration \"Add OpenAI provider\"`\n\n---\n\n## Typical Workflow\n\n### To add a new feature:\n\n```\n1. You implement core logic (most learning)\n2. Agent generates tests\n3. Agent generates docs\n4. Agent optimizes performance\n5. You review & validate\n\n---\n\n## Learning Path\n\n### Week 1: Understand the System\n- [ ] Read `docs/future/skills-prompts-agents.md`\n- [ ] Review all skill files (40 min)\n- [ ] Review all prompts (30 min)\n- [ ] Pick first enhancement\n\n### Week 2: First Enhancement\n- [ ] Implement using skill + prompt\n- [ ] Use agents for tests & docs\n- [ ] Deploy and measure\n- [ ] Document learnings\n\n### Week 3-4: Second Enhancement\n- [ ] Pick Tier 1 use case (easier)\n- [ ] Implement (most learning)\n- [ ] Use agents strategically\n- [ ] Measure business impact\n\n### Month 2: Production Ready\n- [ ] Add monitoring (prompts/add-observability.md)\n- [ ] Optimize (prompts/optimize-performance.md)\n- [ ] Create runbooks\n- [ ] Teach team\n\n---\n\n## File Organization\n\n**Skills** (Best practices - read before implementing):\n```\n.github/skills/llm-integration/SKILL.md\n.github/skills/performance-optimization/SKILL.md\n.github/skills/testing-strategy/SKILL.md\n.github/skills/observability/SKILL.md\n.github/skills/error-handling/SKILL.md\n```\n\n**Prompts** (Step-by-step instructions - follow while implementing):\n```\n/prompts/add-provider.md\n/prompts/add-use-case.md\n/prompts/optimize-performance.md\n/prompts/add-observability.md\n```\n\n**Documentation**:\n```\n/docs/future/skills-prompts-agents.md        👈 Main guide\n/docs/future/agents.md                       Agent descriptions\n/docs/future/use-cases.md                    Future use cases\n```\n\n**Existing code** (Reference for patterns):\n```\n/src/main/java/com/example/aispring/        Spring Boot app\n/docs/*/                                     Existing documentation\n```\n\n---\n\n## FAQ\n\n**Q: Where do I start?**\nA: Read `docs/future/skills-prompts-agents.md` first\n\n**Q: Should I read all skills?**\nA: No. Read the skill relevant to your task.\n\n**Q: Can agents make mistakes?**\nA: Yes.
